# PART 2. 스토리

## 세계관

**시대적 배경: 근미래 생명공학 디스토피아**

인류는 "감정 이식 기술(Emotional Engram Implanting, EEI)"을 완성했다.
원래는 PTSD 치료나 군인 전투력 강화를 위해 개발됐지만, 이 기술은 곧 기업의 손에 넘어갔다.

**ARCANA Corp** — 생체 병기 개발 기업.
포유류 중 가장 빠르게 번식하고, 작고, 인간에게 무해해 보이며, 학습 능력이 뛰어난 **토끼**를 선택해 병기화했다.

> *"두려움을 느끼지 못하는 자가 병기가 된다. 하지만 분노를 품은 자는 그보다 더 강하다."*
> — ARCANA Corp 내부 문서 7조

---

## 주인공: 유닛 VOSS-7

인공 배양된 강화 토끼. 유전자 편집으로 인간 수준의 인지 능력, 반사 신경, 문제 해결 능력 보유.

**심겨진 거짓 기억:**
어릴 적 평화로운 들판, 따뜻한 햇살, 어미와 형제들.
그리고 어느 날 — 총성, 피, 사냥꾼의 웃음소리. 혼자 살아남은 기억.
이 모든 것이 가짜지만 VOSS-7은 그것이 자신의 전부라고 믿는다.

## A 버튼 — 근거리 충전 공격 ✅ 구현 완료

누르고 있으면 충전 게이지가 차오르며, 떼는 타이밍에 따라 단계 결정.
충전 중 원형 범위 미리보기(반투명) 표시.

| 단계 | 충전 시간 | 범위 (반경) | 피해 |
|---|---|---|---|
| I | 0 ~ 0.3초 | 60px (1×) | 20 (1×) |
| II | 0.3 ~ 0.8초 | 72px (1.2×) | 24 (1.2×) |
| MAX | 0.8초 이상 | 90px (1.5×) | 28 (1.4×) |

색상: 초록 계열 (0x4ecca3 → 0x88eecc → 0xffffff)

**치명타** ✅ 구현 완료
- 기본 치명타율 15%, 치명타 피해 ×1.5 (`Player.critRate` / `Player.critMult`)
- 적용 범위: A 근거리 공격 + B 트랩 직격 + 위장 스플래시
- 상태이상(독·화상) 데미지 / 천둥 연쇄 / 적 → 플레이어 데미지는 미적용
- 데미지 숫자: 치명타 시 노란색(#ffdd33) + 굵게 + "!" 접미사
- 영구 해금 "임계 분노" 로 +8% 상승 (15 → 23%)

---

## B 버튼 — 원거리 충전 공격 ✅ 구현 완료

충전 후 발사하면 조준 방향으로 투사체가 날아가며, 적 또는 벽에 닿으면 폭발 소멸.
사거리 제한 없음. 근거리 대비 낮은 위력.

| 단계 | 충전 시간 | 투사체 크기 | 피해 |
|---|---|---|---|
| I | 0 ~ 0.3초 | 14px (r=7) | 10 (0.5×) |
| II | 0.3 ~ 0.8초 | 18px (r=9) | 14 (0.7×) |
| MAX | 0.8초 이상 | 24px (r=12) | 20 (1.0×) |

색상: 빨강 계열 (0xe63946 → 0xff6644 → 0xff2222) / 투사체 속도: 450px/s

**UX 공통 요소**
- 충전 중 게이지 상단 중앙 표시 (단계별 색상 + 라벨)
- 단계 전환 시 게이지 바운스 애니메이션
- 공격 발동 시 원형 확산 이펙트
- 충전 중 범위/궤도 미리보기 (반투명)

## 구역 1 — 육식동물

**여우 (일반) ✅ 구현 완료**
- HP: 30 / 이동 속도: 150px/s / 피해: 8 (접촉, 쿨타임 1초)
- 감지 범위: 250px. 감지 시 플레이어 추적.
- 플레이어 HP 30% 이하 시 도주(2초) 후 재추적. 재도주 방지 유예 1.5초.
- 피격 시 0.5초 경직(빨간 깜빡임). 처치 시 코어 3개 드롭.
- 구현 상태: idle/chase/flee/stun 상태 머신. HP 바 표시. 2~4마리 랜덤 스폰.

**여우 / 족제비 (일반) — 기획안**
- 무리 2~4마리 등장. 한 마리 정면 유인 → 나머지 측면 돌진.
- 체력 30% 이하 도주 후 증원 호출.
- 약점: 도주 직후 0.5초 경직. 3단계 공격으로 무리 일괄 처리 가능.

**매 / 수리 (특수)**
- 원형 궤도 선회 → 급강하 다이브 → 즉시 상승. 지상 공격 회피 불가.
- 약점: 다이브 직전 날개 접기 모션 0.6초. 이때 2~3단계 공격 유효.

**늑대 (엘리트)**
- 주변 소형 적 강화(속도 +20%). 20초마다 울음으로 증원 1회 호출.
- 약점: 울부짖기 모션 중 완전 노출. 우선 제거로 무리 약화.

**FANG (보스) — 대형 검치 맹수**
- 1페이즈: 직선 돌진 / 발 구름 충격파 / 포효 기절
- 체력 50% → 2페이즈: 속도 +40%, 연속 돌진 3회 콤보, 방 전체 충격파 추가
- 약점: 돌진 후 벽 충돌 2초 스턴. 이때 집중 딜.

## 구역 2 — 더 깊은 숲 ✅ 구현 완료

**박쥐 (Bat) — 공중 군집 정찰병** [구역 1 들쥐 대응]
- HP 10 / 속도 130 / 데미지 6 / 코어 1
- 3마리 묶음 스폰. 130~160px 거리에서 좌우로 흔들리며 선회.

- 약점: 비행 패턴이 곡선이라 근접 AoE 조준 난이도 ↑, 단발 데미지에 약함.

**멧돼지 (Boar) — 중량 단일 돌격병** [구역 1 여우 대응]
- HP 55 / 속도 80→270(돌격) / 데미지 18 / 코어 4
- 200px 이내 진입 시 0.5초 예고 → 직선 1.0초 돌격. 벽 충돌 시 1.0초 자기 스턴.
- 돌격 중 장애물(boulder) 파괴 + 돌진 지속 (Fang dash 메커닉 재활용).
- 약점: 돌격 후 0.8초 recover 동안 행동 불가. 측면 회피.

**거미 (Spider) — 지역 차단형** [신규 역할]
- HP 28 / 속도 90 / 데미지 9 / 코어 3
- 3초마다 플레이어 위치+α에 **거미줄 패치**(반경 55px, 지속 4초) 투척.
- 거미줄 위 플레이어: 이동속도 ×0.4 슬로우. 거미 1마리당 활성 2개.
- 약점: 거미 사망 시 거미줄 즉시 페이드. 단일 격파로 슬로우 위협 제거.

**곰 (Bear) — 중량 탱커** [구역 1 고슴도치 대응]
- HP 110 / 속도 75(격노 시 130) / 데미지 22(휘두르기) / 코어 6
- 80px 이내 진입 시 0.5초 예고 → **반경 120px 정면 부채꼴**(180°) 휘두르기. 등 뒤 안전.

**두꺼비 (Toad) — 독 원거리병** [구역 1 다람쥐 대응]
- HP 32 / 속도 65 / 데미지 7(접촉) + 5/s 독 DoT / 코어 3

**검은곰 (BlackBear, 중간 보스) — 8층** [구역 1 늑대 대응]
- HP 280 / 속도 140 / 데미지 24 / 코어 18

- 약점: 포효 1.8초간 완전 정지. 슬램 예고 0.6초간 측면 이탈 가능.

**OWL KING (부엉이왕, 최종 보스) — 10층** [Fang 대응]
- HP 600 / 속도 200(비행) / 데미지 28 / 코어 60
- **공중 강하형 — Fang(지상 돌진) 대비 카메라/회피 축이 다름**

  **Phase 1 (HP 100~60%)** — 공중 정찰
  - shadow_dive: 1.0초 지면 인디케이터 → 직선 강하 380px/s × 0.5s, **강하 중 무적**
  - feather_volley: 0.5초 정지 → 8방향 깃털 투사체 (220px/s, 12 데미지)
  - screech: 0.4초 예고 → 반경 220px AoE, 플레이어 0.6초 기절 (데미지 없음)

### STEP 1. 프로젝트 초기 세팅

```
Phaser 3.60 기반의 탑다운 로그라이크 게임 프로젝트를 세팅해줘.

요구사항:

### STEP 2. 캐릭터 이동 (조이스틱)

```

요구사항:
- 왼쪽 화면 하단에 가상 조이스틱
- 조이스틱 베이스는 고정, 핸들은 반경 50px 이내에서 이동
- 8방향이 아닌 아날로그 방향 (각도 기반)
- 이동 속도: 200px/s
- 터치와 마우스 모두 동작

### STEP 3. 3단계 충전 공격 (핵심 검증)

```
오른쪽 화면 터치/클릭으로 3단계 충전 공격 시스템을 구현해줘.
이게 이 게임의 핵심 메카닉이야.

단계 기준:
- 1단계: 0 ~ 0.3초 → 근거리 타격 (반경 60px, 피해 10)
- 2단계: 0.3 ~ 0.8초 → 중거리 파동 (반경 120px, 피해 20)
- 3단계: 0.8초 이상 → 원거리 관통 (사거리 300px, 너비 40px, 피해 35)

시각 피드백 요구사항:
- 충전 중 캐릭터 앞에 반투명 원/직사각형으로 범위 미리보기 표시
- 충전 게이지를 화면 상단에 표시
- 단계 전환 시 색상 변화 (파랑 → 주황 → 빨강)
- 공격 발동 시 히트 이펙트 (간단한 원형 확산)

조작:
- 오른쪽 화면 어디서나 누르면 충전 시작
- 떼면 해당 단계 공격 발동
- 조이스틱 영역과 겹치지 않게 처리
```

```

방 구조:
- 방 크기: 800x600px (내부 플레이 영역)

방 유형 (이번 구현에서는 전투방만):
- 전투방: 적 2~4마리 랜덤 배치
- 모든 적 처치 시 문 열림
- 처치 전에는 문 잠김

전환 방식:
- 문에 접촉 시 다음 방으로 이동 (페이드 인/아웃)
- 미니맵을 화면 우측 상단에 표시 (작은 격자 형태)
- 현재 위치는 다른 색으로 표시

지형:
- 벽은 캐릭터와 적이 통과 불가
- 방 내부에 간단한 장애물 1~3개 랜덤 배치
```

플레이어 스펙:

### STEP 7. 아이템 시스템

```
패시브 아이템 시스템을 구현해줘.

기본 구조:
- 아이템은 ItemData 배열로 관리 (JSON 형태)
- 각 아이템은 id, name, grade, effect 보유
- 플레이어는 아이템을 최대 6개 보유 가능
- 아이템 효과는 플레이어 스탯에 즉시 적용 (패시브)

우선 구현할 아이템 4개:
1. 예리한 발톱 (일반): 1단계 피해 +15%
2. 두꺼운 털가죽 (일반): 피격 피해 -10%
3. 반동 충전 (레어): 피격 시 충전 게이지 +0.1초
4. 분노 임계값 (에픽): HP 40% 이하 시 전 공격 피해 +35%

드롭:
- 전투방 클리어 시 일반/레어 중 랜덤 드롭
- 드롭된 아이템에 접촉하면 자동 획득
- 획득 시 화면 상단에 아이템 이름 잠깐 표시

HUD에 보유 아이템 아이콘 표시 (하단 중앙)
```

### STEP 8. 보스 구현 (FANG)

```
구역 1 보스 FANG을 구현해줘.

기본 스펙:
- 체력: 500
- 크기: 64x64px (일반 적의 2배)
- HP 바를 화면 하단 중앙에 크게 표시 (보스 전용)

1페이즈 패턴 (HP 100~50%):
1. 직선 돌진: 플레이어 방향으로 400px/s 돌진, 벽에 부딪히면 2초 스턴
2. 발 구름: 제자리에서 충격파 원형 범위 150px 발생, 0.8초 예고 후 발동
3. 포효: 범위 200px 내 플레이어 0.5초 기절

2페이즈 (HP 50% 이하):
- 분노 연출 (빨간색 플래시)
- 속도 +40%
- 돌진 3회 연속 콤보 추가
- 방 전체 충격파 (가장자리 제외) 추가

패턴 전환: 각 패턴은 3~5초 간격으로 랜덤 선택
처치 시: 레어 아이템 1개 드롭 + 코어 50개 드롭
```

코어:

우선 구현할 해금 노드:
- 발톱 강화 I (공격): 1단계 피해 +5%, 코어 20
- 강인한 몸 I (생존): 시작 HP +10, 코어 20
- 코어 수집기 (특수): 드롭 코어 +15%, 코어 45

## 요청 시 꼭 포함할 것들

각 단계를 요청할 때 아래 내용을 함께 명시하면 더 정확한 결과가 나온다.

```
1. Phaser 3.60 버전 기준으로 작성해줘.
2. 기존에 구현된 [파일명]과 연결되어야 해.
3. 터치 입력 우선으로 구현하고 마우스는 대체 수단으로만 써줘.
4. 상수값(속도, 피해량, 타이밍)은 파일 상단에 const로 분리해줘.
   (나중에 밸런싱 조정하기 쉽게)
5. 주석을 한국어로 달아줘.
```

---

## 밸런싱 조정 요청 방법

구현 후 플레이해보고 조정이 필요하면 이렇게 요청한다.

```
[문제 상황]:
3단계 충전 시간 0.8초가 너무 길어서 실전에서 거의 못 씀.

[원하는 변경]:
3단계 기준을 0.8초 → 0.6초로 줄이고,
대신 3단계 피해를 35 → 28로 낮춰줘.
상수로 분리된 값만 변경하면 돼.
```

---

## 디버그 요청 방법

```
[증상]:
여러 마리 적이 동시에 처치될 때 가끔 게임이 멈춤.

[재현 방법]:
3단계 공격으로 4마리 이상 동시 처치 시 발생.

[의심 지점]:
적 처치 시 코어 드롭 + 문 열림 + 방 클리어 체크가
동시에 실행되면서 충돌하는 것 같음.
```

## 주의사항

- **한 번에 하나의 기능만 요청한다.** 여러 기능을 동시에 요청하면 코드가 얽혀서 디버깅이 어렵다.
- **상수는 항상 파일 상단에 분리**해달라고 요청한다. 밸런싱 조정 시 찾기 쉽다.
- **플레이 후 타이밍 검증이 우선**이다. STEP 3 (충전 공격)에서 조작감이 재미없으면 기획을 수정하고 나서 나머지를 진행한다.
- Phaser 버전을 **3.60으로 고정**한다. 버전이 다르면 API가 달라서 오류가 생긴다.

# PART 9. 설계 철학 요약

1. **조작의 단순함, 전략의 깊이** — 버튼 하나의 누르는 시간이 빌드 방향과 전술을 결정.
2. **스토리는 전투 안에 있다** — 보스 대사, EEI 술사 공격, 스토리 방 연출이 서사를 전달.
3. **죽음이 진행이다** — 코어 유지로 매 런이 성장으로 이어짐.
4. **구역이 튜토리얼이다** — 구역 1은 기동력, 구역 2는 우선순위, 구역 3은 패턴 읽기를 자연스럽게 학습.
5. **선택이 엔딩을 만든다** — 휴식 방의 거짓 기억 강화 선택이 누적되어 엔딩 분기에 영향.
