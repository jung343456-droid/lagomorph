# PART 2. 스토리

## 세계관

**시대적 배경: 근미래 생명공학 디스토피아**

인류는 "감정 이식 기술(Emotional Engram Implanting, EEI)"을 완성했다.
원래는 PTSD 치료나 군인 전투력 강화를 위해 개발됐지만, 이 기술은 곧 기업의 손에 넘어갔다.

**ARCANA Corp** — 생체 병기 개발 기업.
포유류 중 가장 빠르게 번식하고, 작고, 인간에게 무해해 보이며, 학습 능력이 뛰어난 **토끼**를 선택해 병기화했다.

> *"두려움을 느끼지 못하는 자가 병기가 된다. 하지만 분노를 품은 자는 그보다 더 강하다."*
> — ARCANA Corp 내부 문서 7조

---

## 코어 (Core)

**정의**

코어(Core)는 이 세계를 구성하는 근원 물질이다. 생명, 기억, 에너지 — 이 세 가지는 코어라는 단일한 물질의 서로 다른 상태에서 비롯된다. 코어는 단순한 자원이 아니라 존재의 조건 그 자체다.

> *"우리는 코어로 태어나고, 코어로 생각하고, 코어로 돌아간다."*
> — 코어 발견 이전 시대의 고대 격언. ARCANA는 이것이 비유가 아니라 사실임을 증명했다.

**기원**

코어는 인류가 발명한 것이 아니다. 지각 깊은 곳, 생명체의 신경계, 오래된 숲의 토양 — 자연 어디에나 극미량으로 존재해왔다. 고대부터 일부 생명체는 코어 밀도가 높은 장소에 본능적으로 모여들었고, 인류는 그것을 "성지" 혹은 "기운이 좋은 땅"이라고 불렀다.

코어가 과학적으로 발견된 것은 약 80년 전. 초기에는 단순한 에너지원으로 여겨졌지만, 연구가 깊어질수록 코어가 기억과 감정의 물리적 기반임이 밝혀졌다. 생명체가 경험하는 모든 감정은 신경계 내 코어 입자의 진동 패턴으로 저장된다. **기억이란 코어의 결정 구조다.**

ARCANA Corp은 이 발견 직후 코어 연구소 전체를 인수했다. 특허, 채굴권, 연구 인력까지 모두. 현재 코어의 채굴, 정제, 유통은 사실상 ARCANA의 독점이다.

## 주인공: VOSS 시리즈

인공 배양된 강화 토끼 병기. 유전자 편집으로 인간 수준의 인지 능력, 반사 신경, 문제 해결 능력을 보유한다.
각 개체에는 일련번호가 매겨진다 — VOSS-1, VOSS-2, … 현재 가동 중인 개체는 **VOSS-7**(코드명 soma).

**로그라이크 = 서사 그 자체**
한 번의 플레이는 한 개체의 한 사이클(훈련 주기)이다. 개체가 쓰러지거나 사이클을 마치면, ARCANA는
다음 번호의 새 개체를 깨운다. 새 개체는 **같은 거짓 기억을 안고** 처음부터 다시 시작하며, 번호는 계속
증가한다. 플레이어가 마주하는 "다음 판"은 곧 다음 VOSS 개체의 생애다.

**심겨진 거짓 기억**
어릴 적 평화로운 들판, 따뜻한 햇살, 어미와 형제들. 그리고 어느 날 — 총성, 피, 사냥꾼의 웃음소리.
혼자 살아남은 기억. 이 모든 것이 가짜지만, 개체는 그것이 자신의 전부라고 믿고 **사냥꾼에 대한 복수**를
유일한 목표로 삼는다.

---

## 사냥꾼의 정체 — 핵심 반전

VOSS의 거짓 기억 속 "사냥꾼"은 실존하는 적이 아니다. ARCANA가 VOSS 개체를 한계까지 단련시키기 위해
만든 **훈련용 로봇(구조체)**이고, 던전 전체가 ARCANA의 훈련 프로그램이다.

- 구역 1~2(숲)의 동물형 적부터 구역 3~4의 동물·**인간 사냥꾼**까지, 모두 VOSS를 단련시키는 훈련
  구조체다. 자연 생태계처럼 보이는 필드는 ARCANA가 구축한 **훈련장(시뮬레이션 환경)**일 뿐이다.
- 심어진 복수심은 개체가 두려움 없이, 한계까지 싸우게 만드는 동기 장치다. "사냥꾼을 죽인다"는 목표가
  개체를 구역 4 끝까지 몰아간다.
- **구역 3** 보스(사냥꾼)를 마침내 쓰러뜨려도 복수의 감정은 채워지지 않고 **공허함**만 남는다.
- **구역 4** 보스 — 추격의 끝에서 쓰러뜨린 사냥꾼이 **기계(로봇)**였음을 깨닫는다. 복수의 대상이
  처음부터 사람이 아니었다는 사실이 거짓 기억에 결정적 균열을 낸다.

---

## 진실을 향한 여정

사냥꾼이 로봇임을 깨달은 개체는, 복수가 아닌 **진실**을 좇기 시작한다.
훈련장 너머에는 자신을 만들고 조종해 온 **ARCANA 시설**이 있다. 여기서부터 개체는 시뮬레이션이 아닌
실제 위협과 마주하며, 자신의 기억과 정체성의 진실(EEI 이식의 증거, 자신이 몇 번째 개체인지)에 다가간다.

> 게임 진행 큰 흐름: 숲에서의 복수(구역 1~2) → 사냥꾼과의 결착(구역 3~4) → 사냥꾼=로봇 자각 →
> ARCANA 시설에서의 진실 추적(엔드게임).

## 엔딩 분기

> 각 개체의 "다음 번호 각성"은 ARCANA가 의도한 정상 순환이다. 엔딩은 이 개체가 그 순환을
> **반복할 것인가, 끊을 것인가**의 갈림길이다.

**엔딩 A — "완성된 병기" (레드)**
분노를 ARCANA로 돌려 시설을 파괴한다. 그러나 백업 배양조에서 VOSS-8이 눈을 뜨고, 같은 거짓 기억을
안은 채 순환은 계속된다.
> *"복수는 복수를 낳는다."*

**엔딩 B — "최초의 선택" (화이트)**
분노를 내려놓고 EEI 데이터를 외부에 유출한 뒤 스스로 종료한다. 순환을 잇기를 거부함으로써 비로소
순환을 끊는다. 크레딧 중 생체 병기 스캔들이 세상에 알려진다.
> *"나는 기억이 없다. 그러므로 나는 처음으로 자유롭다."*

**히든 엔딩 — "0번째 유닛" (골드)**
숨겨진 조건 달성 시. 세뇌에 실패한 최초 개체 VOSS-0이 시설 어딘가에 살아있음을 발견.

## A 버튼 — 근거리 충전 공격 ✅ 구현 완료

누르고 있으면 충전 게이지가 차오르며, 떼는 타이밍에 따라 단계 결정.
충전 중 원형 범위 미리보기(반투명) 표시.

| 단계 | 충전 시간 | 범위 (반경) | 피해 |
|---|---|---|---|
| I | 0 ~ 0.3초 | 60px (1×) | 20 (1×) |
| II | 0.3 ~ 0.8초 | 72px (1.2×) | 24 (1.2×) |
| MAX | 0.8초 이상 | 90px (1.5×) | 28 (1.4×) |

색상: 초록 계열 (0x4ecca3 → 0x88eecc → 0xffffff)

**치명타** ✅ 구현 완료
- 기본 치명타율 15%, 치명타 피해 ×1.5 (`Player.critRate` / `Player.critMult`)
- 적용 범위: A 근거리 공격 + B 트랩 직격 + 위장 스플래시
- 상태이상(독·화상) 데미지 / 천둥 연쇄 / 적 → 플레이어 데미지는 미적용
- 데미지 숫자: 치명타 시 노란색(#ffdd33) + 굵게 + "!" 접미사
- 영구 해금 "임계 분노" 로 +8% 상승 (15 → 23%)

---

## B 버튼 — 원거리 충전 공격 ✅ 구현 완료

충전 후 발사하면 조준 방향으로 투사체가 날아가며, 적 또는 벽에 닿으면 폭발 소멸.
사거리 제한 없음. 근거리 대비 낮은 위력.

| 단계 | 충전 시간 | 투사체 크기 | 피해 |
|---|---|---|---|
| I | 0 ~ 0.3초 | 14px (r=7) | 10 (0.5×) |
| II | 0.3 ~ 0.8초 | 18px (r=9) | 14 (0.7×) |
| MAX | 0.8초 이상 | 24px (r=12) | 20 (1.0×) |

색상: 빨강 계열 (0xe63946 → 0xff6644 → 0xff2222) / 투사체 속도: 450px/s

**UX 공통 요소**
- 충전 중 게이지 상단 중앙 표시 (단계별 색상 + 라벨)
- 단계 전환 시 게이지 바운스 애니메이션
- 공격 발동 시 원형 확산 이펙트
- 충전 중 범위/궤도 미리보기 (반투명)

### 올가미 덫 (Snare) — 구역 3 시그니처 군중제어

- 활 사냥꾼·사냥꾼 보스가 바닥에 덫을 설치. 플레이어 center가 반경 ≈40px 진입 시 **루트(이동 0) ~1.0s**.
- **끊어내기(활 사냥꾼·사냥꾼 보스 공통)**: 덫 범위 안에서 근거리 공격(A)을 5회 하면 덫이 끊어져 즉시 해제(덫 제거 + `player.clearRoot`).
  진행도는 덫 알파가 옅어지며 표시. (`SNARE_BREAK_HITS=5`, `BowHunter._onPlayerAttack`·`HunterBoss._onPlayerAttack`)
- 구현: 거미줄·독 웅덩이의 **잔존 hazard 패턴 재사용**(`registerLingeringHazard`/`tickLingering`/
  `disposeHazards`). 단 `player.applySlow`(속도 ×0.3) 대신 신규 **`player.applyRoot(dur)`**(속도 0) 호출.
- 설치자당 활성 1~2개 제한(거미줄 `WEB_MAX` 패턴). 텍스처: 신규 `snare`(net/올가미).

격자형 방 생성 시스템을 구현해줘. Binding of Isaac 스타일이야.

방 구조:
- 방 크기: 800x600px (내부 플레이 영역)
- 격자: 최대 8x6 그리드에 9~12개 방 랜덤 배치
- 방 연결: 상하좌우 문으로 연결
- 시작방은 항상 중앙 부근에 배치

방 유형 (이번 구현에서는 전투방만):
- 전투방: 적 2~4마리 랜덤 배치
- 모든 적 처치 시 문 열림
- 처치 전에는 문 잠김

전환 방식:
- 문에 접촉 시 다음 방으로 이동 (페이드 인/아웃)
- 미니맵을 화면 우측 상단에 표시 (작은 격자 형태)
- 현재 위치는 다른 색으로 표시

지형:
- 벽은 캐릭터와 적이 통과 불가
- 방 내부에 간단한 장애물 1~3개 랜덤 배치
```

### STEP 7. 아이템 시스템

```
패시브 아이템 시스템을 구현해줘.

기본 구조:
- 아이템은 ItemData 배열로 관리 (JSON 형태)
- 각 아이템은 id, name, grade, effect 보유
- 플레이어는 아이템을 최대 6개 보유 가능
- 아이템 효과는 플레이어 스탯에 즉시 적용 (패시브)

우선 구현할 아이템 4개:
1. 예리한 발톱 (일반): 1단계 피해 +15%
2. 두꺼운 털가죽 (일반): 피격 피해 -10%
3. 반동 충전 (레어): 피격 시 충전 게이지 +0.1초
4. 분노 임계값 (에픽): HP 40% 이하 시 전 공격 피해 +35%

드롭:
- 전투방 클리어 시 일반/레어 중 랜덤 드롭
- 드롭된 아이템에 접촉하면 자동 획득
- 획득 시 화면 상단에 아이템 이름 잠깐 표시

HUD에 보유 아이템 아이콘 표시 (하단 중앙)
```

### STEP 8. 보스 구현 (FANG)

```
구역 1 보스 FANG을 구현해줘.

기본 스펙:
- 체력: 500
- 크기: 64x64px (일반 적의 2배)
- HP 바를 화면 하단 중앙에 크게 표시 (보스 전용)

1페이즈 패턴 (HP 100~50%):
1. 직선 돌진: 플레이어 방향으로 400px/s 돌진, 벽에 부딪히면 2초 스턴
2. 발 구름: 제자리에서 충격파 원형 범위 150px 발생, 0.8초 예고 후 발동
3. 포효: 범위 200px 내 플레이어 0.5초 기절

2페이즈 (HP 50% 이하):
- 분노 연출 (빨간색 플래시)
- 속도 +40%
- 돌진 3회 연속 콤보 추가
- 방 전체 충격파 (가장자리 제외) 추가

패턴 전환: 각 패턴은 3~5초 간격으로 랜덤 선택
처치 시: 레어 아이템 1개 드롭 + 코어 50개 드롭
```

### STEP 9. 코어 및 영구 해금

```

코어:

- 해금 조건: 이전 노드 해금 완료 + 코어 충족

우선 구현할 해금 노드:
- 발톱 강화 I (공격): 1단계 피해 +5%, 코어 20
- 강인한 몸 I (생존): 시작 HP +10, 코어 20
- 코어 수집기 (특수): 드롭 코어 +15%, 코어 45

## 요청 시 꼭 포함할 것들

각 단계를 요청할 때 아래 내용을 함께 명시하면 더 정확한 결과가 나온다.

```
1. Phaser 3.60 버전 기준으로 작성해줘.
2. 기존에 구현된 [파일명]과 연결되어야 해.
3. 터치 입력 우선으로 구현하고 마우스는 대체 수단으로만 써줘.
4. 상수값(속도, 피해량, 타이밍)은 파일 상단에 const로 분리해줘.
   (나중에 밸런싱 조정하기 쉽게)
5. 주석을 한국어로 달아줘.
```

---

## 밸런싱 조정 요청 방법

구현 후 플레이해보고 조정이 필요하면 이렇게 요청한다.

```
[문제 상황]:
3단계 충전 시간 0.8초가 너무 길어서 실전에서 거의 못 씀.

[원하는 변경]:
3단계 기준을 0.8초 → 0.6초로 줄이고,
대신 3단계 피해를 35 → 28로 낮춰줘.
상수로 분리된 값만 변경하면 돼.
```

## 디버그 요청 방법

```
[증상]:
여러 마리 적이 동시에 처치될 때 가끔 게임이 멈춤.

[재현 방법]:
3단계 공격으로 4마리 이상 동시 처치 시 발생.

[의심 지점]:
적 처치 시 코어 드롭 + 문 열림 + 방 클리어 체크가
동시에 실행되면서 충돌하는 것 같음.
```

## 주의사항

- **한 번에 하나의 기능만 요청한다.** 여러 기능을 동시에 요청하면 코드가 얽혀서 디버깅이 어렵다.
- **상수는 항상 파일 상단에 분리**해달라고 요청한다. 밸런싱 조정 시 찾기 쉽다.
- **플레이 후 타이밍 검증이 우선**이다. STEP 3 (충전 공격)에서 조작감이 재미없으면 기획을 수정하고 나서 나머지를 진행한다.
- Phaser 버전을 **3.60으로 고정**한다. 버전이 다르면 API가 달라서 오류가 생긴다.

# PART 9. 설계 철학 요약

1. **조작의 단순함, 전략의 깊이** — 버튼 하나의 누르는 시간이 빌드 방향과 전술을 결정.
2. **스토리는 전투 안에 있다** — 보스 대사, EEI 술사 공격, 스토리 방 연출이 서사를 전달.
3. **죽음이 진행이다** — 코어 유지로 매 런이 성장으로 이어짐.
4. **구역이 튜토리얼이다** — 구역 1은 기동력, 구역 2는 우선순위, 구역 3은 패턴 읽기를 자연스럽게 학습.
5. **선택이 엔딩을 만든다** — 휴식 방의 거짓 기억 강화 선택이 누적되어 엔딩 분기에 영향.
